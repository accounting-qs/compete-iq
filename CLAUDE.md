# Rules

- Only commit when asked. Never push to remote unless explicitly requested.
- Do not modify existing features, UI, or settings unless (a) the user explicitly asks for that change, or (b) it is part of a plan the user has approved. This includes copy, styling, layout, behavior, config values, and defaults — even if they look inconsistent or "wrong." If a change outside the current task seems needed, surface it as a suggestion and wait for approval instead of editing.
- Before every Edit/Write, confirm: is this file/change in the task the user asked for, or in an approved plan? If not, stop and ask. Scope creep requires explicit permission, not inference.
